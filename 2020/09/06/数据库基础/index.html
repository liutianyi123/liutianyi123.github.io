Chapter 1 Database System

为什么要选择数据库

一方面：当今世界数据无处不在(ubiquitous)具有普遍性(pervasive)

另一方面：数据库存储数据具有安全性(in a secure way)

数据和信息的关系

① 数据由原始的信息(raw facts)构成

② 信息来源于对原始数据(raw data)的处理

数据库简介

数据库是一种存储数据(stores data)的计算机结构(computer structure)具有共享性(Shared)和集成性(integrated)

数据库管理系统DBMS(Database management system)是一种管理数据库的结构(Manages the database structure)，并控制数据的访问(Controls access to data)还是一种程序的收集(Collection of programs)

数据库管理系统的作用和优势

数据库管理系统是用户于数据库之间的桥梁(intermediary)

① 实现数据的共享(sharing)

② 高效的管理数据(effective data management)

③ 向用户展现数据视图(view of data)

④ 改进共享、安全、集成、访问(access)、决策制定(decision-making)、生产力(productivity)等。

数据库种类

① 通过支持用户(user)数量划分

一次支持一个用户：单用户数据库(Single-user database)

​ eg.桌面数据库(Desktop database):个人计算机上的单用户数据库

一次支持多个用户：多用户数据库(Multiuser database)

​ eg.工作组数据库(Workgroup databases):支持少数用户或特定部门

​ 企业数据库(Enterprise database):支持多个部门的多个用户

② 通过位置(locaction)分类

集中式数据库(Centralized database):数据位于单一站点(a single site)

分布式数据库(Distributed database):数据分布在不同的站点(different sites)

云数据库(Cloud database):使用云数据服务(cloud data services)创建和维护

③ 通过数据类型(data type)分类

通用数据库(General-purpose database):包含在多个学科(multiple disciplines)中使用的各种数据

特定学科数据库(Discipline-specific database):包含特定学科领域(specific subject areas)的数据

运营数据库(Operational database):用于支持公司的日常运营(day-to-day operations)

​ eg.事务数据库(Transactional database)

分析数据库( Analytical database):存储专门用于战术(tactical)或战略决策制定(trategic decision making)的历史数据(historical data)和业务指标(business metrics)

​ eg.数据仓库(Data warehouse):以用于决策支持的优化格式(a format optimized for decision support)存储数据

​ 联机分析处理(Online analytical processing (OLAP)):数据仓库检索(retrieving)、处理(processing)和建模(modeling)数据的工具

​ 商业智能(商业智能):捕获和处理(captures and processes)商业数据，以生成支持决策制定的信息

④ 通过结构化程度(Degree of structure)

非结构化数据:以其原始状态(original state)存在

结构化数据:源于格式化(formatting)

半结构化数据:在一定程度上经过处理(process)

注意：NoSQL(不仅仅是SQL)主要处理非结构化数据

为什么数据库设计很重要

良好的数据库:便于数据管理(data management)，生成准确和有价值的信息

不良的数据库:导致错误和糟糕的决策(poor decision making)

文件系统数据处理的发展

手册的文件系统(Manual file systems):通过文件夹(file folders)
和文件柜(filing cabinets)系统管理的数据

计算机化的文件系统(Computerized file systems):基于计算机的系统( computerbased system)来跟踪数据(track data)并生成所需的报告

文件系统再现(File system redux):现代终端用户生产力工具,包括电子表格程序，如Excel

文件系统数据(File System Data)处理问题

开发时间长(Lengthy development times)

难以快速得到答案

复杂的系统管理(Complex system administration)

缺乏安全性(Lack of security)，数据共享(data sharing)受限

结构依赖

① 对文件的访问取决于它自身的结构(own structure)

② 所有文件系统程序都被修改以符合新的文件结构(conform to a new file structure)

结构独立

① 在不影响应用程序访问数据(the application’s ability to access the data)的情况下更改文件结构

② 结构独立性很重要，因为它大大减少了编程工作(programming effort )和程序维护成本(program maintenance costs)。

数据依赖

当数据存储特性(data storage characteristics)发生变化时，数据访问(Data access)也发生变化

数据独立

在不影响程序访问该数据的能力的情况下改变数据存储特性(Data storage characteristic)

总而言之：数据依赖的实际意义在于逻辑(logical)和物理格式(physical format)的区别

数据冗余(Data Redundancy)

描述：将相同的数据存储在不同的地方

  • 信息岛(Islands of information),即分散的(scattered)数据位置

  • 增加相同数据的不同版本的概率

结果：

① 数据安全性降低(Poor data security)

增加未经授权(unauthorized)查阅资料副本(a copy of the data)的机会

② 数据不一致(Data inconsistency)

当相同数据的不同版本和冲突版本出现在不同的地方时存在

③ 数据输入错误(Data-entry errors)

当在几个不同的文件中创建复杂的条目时，更有可能发生

④ 数据完整性问题(Data integrity problems)

保证数据的准确性和一致性->需要手动的努力

数据异常(Data Anomalies)

描述：没有完成所有必需的更改时进行开发

类别：

  • 更新异常

  • 插入异常

  • 删除异常

数据库系统(Database System)

描述：逻辑相关的数据存储在一个逻辑数据存储库(logical data repository)中

  • 分布在多个存储设施之间
  • DBMS消除了大多数文件系统的数据不一致(data inconsistency)、数据异常(data anomaly)、数据依赖(data dependence)和结构依赖(structural dependence)问题

数据库系统环境

数据库系统就是在数据库环境中定义(define)和规范(regulate)数据的收集(collection)、存储(storage)、管理(management)和使用(use of data)的组件的组织

DBMS功能

① 数据字典管理(Data dictionary management)

​ 数据字典:存储数据元素(data elements)及其关系的定义

② 数据存储管理(Data storage management)

​ 确保高效的性能

③ 数据转换和演示(Data transformation and presentation)

​ 数据的格式符合(conform to)逻辑预期(logical expectations)

④ 安全管理(Security management)

​ 加强用户安全和数据隐私(data privacy)

⑤ 多用户访问控制(Multiuser access control)

​ 确保多个用户可以同时访问数据库而不影响其完整性

⑥ 备份和恢复管理(Backup and recovery management)

​ 能够在失败后恢复数据库

⑦ 数据完整性管理(Data integrity management)

​ 最小化冗余，最大化一致性

⑧ 数据库访问语言和应用程序编程接口(Database access languages and application programming interfaces)

​ 查询语言:让用户指定必须做什么，而不必指定如何做

​ 结构化查询语言(SQL):事实上的查询语言和数据访问标准，被大多数DBMS供应商支持

⑨ 数据库通信接口(Database communication interfaces)

​ 通过多个、不同的网络环境接受终端用户请求

管理数据库系统:焦点转移

数据库系统的缺点：

① 成本高(Increased acquisition and operating costs)

需要精密的硬件和软件(sophisticated hardware and software)以及高度熟练的人员(highly skilled personnel)

② 管理复杂(Management complexity)

许多不同的技术接口(interface with many different technologies)

③ 保持时效性(Maintaining currency)

为了最大化数据库系统的效率

④ 依赖供应商(Vendor dependence)

需要在技术和人才培训方面进行投资

⑤ 更新与更换频繁(Frequent upgrade/replacement cycles)

总结

  • 数据由原始事实组成，通常存储在数据库中

  • 数据库设计定义了数据库结构，可以根据用户数量、位置以及数据使用和结构进行分类吗

  • 数据库从手工和计算机化的文件系统演变而来，因为文件系统数据管理有一些限制所以开发了DBMS

Review Questions -L01

答案来源于：传送门

1.Define each of the following terms:data,field,record,file

Data: raw facts
Field: an attribute, same data type
Record: related data that describes an entity(描述一个实体的相关数据)
File: a collection of related records.(相关记录的集合)

2.What is data redundancy, and which characteristics of the file system can lead to it?

Data redundancy occurs when the same data are stored in multiple places unnecessarily.The use of tables in different parts of the organization can cause it.

3.What is data independence, and why is it lacking in file systems?

Data independence exists when you can change the data storage characteristics without impeding the program’s ability to access the data.

Any program that accesses system’s file must tell the computer not only what to do, but how to do it. Consequently, each program must contain lines that specify the opening of a specific type of file, its record specification, and its field definitions. Data dependence makes the file system extremely cumbersome(繁琐) from the point of view of a programmer and database manager.

4.What is a DBMS, and what are its functions?

DBMS = Database Management System

a collection of programs that manages the database structure and controls access to the data stored in the database.

functions of a DBMS:

data dictionary management
data storage management
data transformation and presentation
security management
multiuser access control
backup and recovery management
data integrity management
database access languages and application programming interfaces.
database communication interfaces.

5.What is structual independence, and why is it important?

When it is possible to make changes in the file structure without affecting the application program’s ability to access the data.

6.Explain the differences between data, information, and a database

Data are raw facts that have not been processed.

Information is data that has been processed.

Databases help to facilitate the manipulation of data to create information by providing an organizational structure that makes relationships and connections between data explicit.（数据库通过提供使数据之间的关系和连接明确化的组织结构，有助于促进数据创建信息的操作。）

7.What is the role of a DBMS, and what are its advantages?

The intermediary between the user and the database.

advantages:

improved data sharing
improved data security
better data integration
minimized data inconsistency
improved data access
improved decision making
increased end-user productivity

disadvantages:

increased costs
management complexity
maintaining currency
vendor dependence
frequent upgrade/replacement cycles

8.List and describe the different types of databases.

  • single user database: supports only one user at a time.

  • desktop database: a single user database that runs on a personal computer

  • multi-user database: supports multiple users at one time.

  • work-group database: a multi-user database that supports a small number of users

  • enterprise database: a multi-user database that supports a large number of users

  • centralized database: supports data located at a single site.

  • distributed database: supports data distributed across several different sites

  • operational/transactional/production database: for day-to-day operations

  • data warehouse: stores data used to generate info required to make tactical or strategic decisions

9.What are the main components of a database system?

hardware,software,people,procedures,data

10.What is metadata(元数据)?

Can integrate and manage end user data.

11.Explain why database design is important.

a poorly designed database may produce difficult to trace errors that result in bad decision making – and bad decision making can lead to the failure of an organization.

12.What are the potential costs of implementing a database system?

  • sophisticated hardware and software, trained personnel

  • training, licensingm and regulation compliance costs

  • vendor dependence - vendors are less likely to offer pricing point advantages to existing customers

  • updating of hardware and software

13.Use examples to compare and contrast unstructured and structured data. Which type is more prevalent in a typical business environment?

examples of both types would include:

An invoice. If one were to take an invoice and simply scan it into a graphic, it would be unstructured data. In contrast, if it were processed and put into a database (subsequently becoming structured data), employees could eventually find the monthly averages, amount owed, etc. from various invoices.

While both are prevalent, I would think semi-structured data would be the most common in a typical business. Some data is stored but not processed (unstructured data such as memos), and some others are stored in databases (such as invoices) but most data are only processed to a certain extent that is displayed in a prearranged format but not able to yield all of the information contained within.

14.What are some basic database functions that a spreadsheet cannot perform.

support for self-documentation through metadata enforcement of data types or domains to ensure consistency of data within a column, defined relationships among tables, or contraints to ensure consistency of data across related tables.( 通过强制执行数据类型或域的元数据来支持自文档化，以确保列内数据的一致性，表之间已定义的关系或约束以确保跨相关表的数据一致性)

15.What common problems do a collection of spreadsheets created by end users share with the typical file system?

lengthy development times
difficulty of getting quick answers
complex system administration
lack of security and limited data sharing
extensive programming

16.Explain why the cost of ownership may be lower with a cloud database than with a traditional, company database.

Because you are outsourcing your data to a entity with large database assets, thus lowering costs. Also you eliminate costs in hardware and software required to maintain your own database.（因为您正在将您的数据外包给具有大型数据库资产的实体，从而降低成本。此外，您还消除了维护您自己的数据库所需的硬件和软件成本）
